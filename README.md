### 系统调用

> 系统调用的前置函数：reentersyscall 
 
主要动作：
1. 标记栈抢占请求； 
2. 禁止栈拆分； 
3. 保存现场（PC、SP和G）；
4. 栈追踪；
5. 唤醒等待的M；
6. 安全节点检查避免数据竞争；
7. 把当前P放到m.oldp里；
8. 解除M和P的绑定；
9. 把P的状态设置成系统调用(_Psyscall)；
10. 判断是否开启GC，如果GC开启则把当前的P进入停止，如果当前的P是最后一个P时则运行GC，P的数量默认是系统核数
